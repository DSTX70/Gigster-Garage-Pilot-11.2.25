But for PR checklists, those belong in GitHub—that’s where reviews, CI, and tracking happen.

Here’s the clean setup so you don’t have to copy/paste from chat again:

What to do now (super quick)

Add a PR template to your repo (so every new PR auto-loads the checklist UI):

Create this file in your repo: .github/pull_request_template.md

Paste the template below (it includes owner sections you can keep/remove per PR)

Push from Replit (Replit → Git → Commit → Push).

Open the PR on GitHub and the template will be pre-filled. Keep the section that matches the owner (Forge, Pulse, etc.), delete the rest, and submit.

Drop-in file: .github/pull_request_template.md
